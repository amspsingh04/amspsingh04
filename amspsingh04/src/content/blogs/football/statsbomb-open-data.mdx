## Why open data first

Before touching proprietary feeds, open datasets let you standardise a workflow: ingest events, validate schema, aggregate to player / team level, and plot without worrying about licensing. StatsBomb’s open release is a natural on-ramp for xG models, pass networks, and defensive pressure maps.

## Loading events (Python sketch)

Below is a minimal pattern: fetch or read JSON, flatten to a dataframe, and inspect core columns (`type`, `location`, `shot`, etc.). Swap in the real path or API you use for your environment.

```python
import pandas as pd
from pathlib import Path

# Example: after downloading StatsBomb open JSON for one competition
DATA = Path("data/statsbomb_open")
events = pd.read_json(DATA / "events.json", lines=False)

# Normalise list-like columns in a real pipeline; here we only peek
cols = ["id", "index", "period", "timestamp", "type", "possession", "location"]
available = [c for c in cols if c in events.columns]
preview = events[available].head(3)
print(preview.to_string(index=False))
```
