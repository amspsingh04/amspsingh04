Football analysis is basically a **question-driven retelling of a match**: tracking the events that happened, and the events that no one saw. There is a lot of data that can get generated in every match—the events that happen on the ball, off it, coordination between a subsection of the team’s formations, and the battles that take place between each lineup. There is a unique importance to all of this.


On-the-ball events let us understand **possession patterns**, **shots**, and overall scoring-style metrics — metrics which are of importance to a person who watches football on fotmob. Off-the-ball tracking lets us understand **sustained pressure**, **possible threats in play**, and it gives us an overall understanding of *why* an on-the-ball event would have happened. Every single on-the-ball event has at least four or five off-the-ball events that serve as its context—maybe that is part of why that kind of data is paid. Lineup data lets us understand the **shapes and patterns** behind a team’s attacks, defences, and transitions; that gives us a big-picture read on what is likely to happen throughout the match.


## So what analyses can we perform, and what questions do we have to ask?

The most basic layer is to **reconstruct the match timeline**—goals, fouls, corners, offsides. Going a little deeper, we can understand **possessions** and the key phases of offence and defence.

If we ask the right questions, we can also understand **transitional phases**, **build-up phases**, and the **role and behaviour** of each player in the lineup. From there we can evaluate a player’s **passing profile**, **progression metrics**, **actions**, and **role adherence**. Combining it all, we can analyse the **whole team** and the **whole game**.

There is a lot we can question, and a lot of it will be asked and answered over this blog series.

## StatsBomb open data—and where we start

Let us understand what the **StatsBomb open data** events are all about, and what we can do with them, taking the example of the **La Liga** match between **Real Madrid** and **Barcelona** on **2017-12-23**, at the **Santiago Bernabéu**. In the events data subdirectory of the StatsBomb open-data git repo, this match is **`9736.json`**.

### File, helpers, and loading events

```python
file = Path("./9736.json")
events = json.loads(file.read_text(encoding="utf-8"))

def match_clock(event):
    return int(event.get("minute", 0)) * 60 + int(event.get("second", 0))

def ev_team(ev):
    return ev.get("possession_team", {}).get("name") or ev.get("team", {}).get("name")

def ev_poss(ev):
    return ev.get("possession")
def is_goal_event(ev):
    return (
        ev.get("type", {}).get("name") == "Shot"
        and ev.get("shot", {}).get("outcome", {}).get("name") == "Goal"
    )

goal_indices = [i for i, ev in enumerate(events) if isinstance(ev, dict) and is_goal_event(ev)]

all_keys = sorted({k for ev in events if isinstance(ev, dict) for k in ev.keys()})
len(events), all_keys
```

So first things first, we can reconstruct the match timeline—we ask the simple question *“What all happened during the match?”*, and the answer can be found here - we just find the shots that were goals, saved, or went wayward, and some passes or actions which were offside, and corners. The code for that is like this:
```python
rows = []
for ev in events:
    t = ev.get("type", {}).get("name")     ## find the type of event which was happening
    team = ev.get("possession_team", {}).get("name")  ## which team 
    player = ev.get("player", {}).get("name")     ## who was the player
    clock = match_clock(ev)       ## when
    
    ## here we can see the events such as different kinds of shots, corners, offsides, etc.
    if t == "Shot":
        rows.append((clock, team, ev.get("shot", {}).get("outcome", {}).get("name")+" shot", player))

    if t == "Pass" and ev.get("pass", {}).get("type", {}).get("name") in ["Corner", "Pass Offside"]:
        rows.append((clock, team, ev.get("pass", {}).get("type", {}).get("name"), player))

    if t == "Offside":
        rows.append((clock, team, "Offside", player))
  
## arrange that and show it in a dataframe format  
timeline = pd.DataFrame(rows, columns=["clock_s", "team", "event", "player"])
timeline.sort_values("clock_s").head(3)
```

I think we should now dig deeper and find out what was the possession pattern for the teams, and what events influenced the passage of play. The simplest way to do that would be to look at who had a greater share of possession at each point of time, and what event happened when.
That would be quite a few blocks of code, so let me describe it first; From the event feed you rebuild each team’s possession time in seconds, turn that into a running share of the ball by match minute, then plot those shares and mark goal times.

```python
df = pd.DataFrame([{
    "clock_s": match_clock(ev),
    "possession": ev.get("possession"),
    "poss_team": (ev.get("possession_team") or {}).get("name"),
} for ev in events])
df = df.dropna(subset=["clock_s", "possession", "poss_team"]).sort_values("clock_s")

intervals = (df.groupby(["possession", "poss_team"], as_index=False)
              .agg(start=("clock_s", "min"), end=("clock_s", "max")))[["poss_team", "start", "end"]]
teams = sorted(intervals["poss_team"].unique())

def overlap_0_t(start, end, t):
    return max(0, min(end, t) - max(start, 0))

match_end = int(df["clock_s"].max())
rows = []
for t in range(60, match_end + 60, 60):
    team_time = {
        tm: sum(overlap_0_t(s, e, t) for s, e in intervals.loc[intervals["poss_team"] == tm, ["start", "end"]].itertuples(index=False, name=None))
        for tm in teams
    }
    tot = sum(team_time.values()) or 1.0
    rows.append({"minute": t // 60, **{tm: 100.0 * team_time[tm] / tot for tm in teams}})

running = pd.DataFrame(rows)

goals = timeline[timeline["event"] == "Goal shot"].copy()

if "minute" not in goals.columns:
    goals["minute"] = goals["clock_s"] / 60.0

fig, ax = plt.subplots(figsize=(12,4))

for team in teams:
    ax.plot(running["minute"], running[team], label=team)

# goal markers
for _, g in goals.iterrows():
    team = g["team"]
    ax.axvline(g["minute"], linestyle="--", alpha=0.7, linewidth=1)

ax.set_ylim(0, 100)
ax.set_ylabel("Running possession % (cumulative)")
ax.set_xlabel("Minute")
ax.legend()
ax.grid(True, alpha=0.3)
plt.show()
```

Now, here it looks like the first goal was on the back of sustained pressure over a long passage of play, the second seems to come against the run of play, and the last one seems to be on the back of Barca controlling the game in the final 15-20 minutes. But can we validate that somehow? 
Let us try to see the 200 events before each goal, and see if that tells us something. We can create a window of events before the goal, and take each event and give them a threat score. Then we can analyse how the threat fluctuated, how the progress happened, and what was exactly transpiring on the ground.
